# **线程池**
> 池式结构，维持和管理固定数量的线程

- 维持：避免频繁的创建和销毁线程（存在用户态到内核态的系统调用开销）
- 管理：在多线程环境中，涉及临界资源以及线程状态的管理

- 为什么需要线程池？
> 为了优化性能，防止过度地占用某个线程
> - 复用线程资源
> - 充分利用系统资源
> - 异步执行耗时任务，比如磁盘io、网络io、cpu密集型任务等等


- 线程池线程数量如何确定？
> - CPU密集型：CPU核心数
> - IO密集型：2 * CPU核心数 + 2

- 线程池的模型：生产者-消费者模型
> - 生产者线程生产任务，投放在队列中，消费者线程从队列中取出任务并执行
